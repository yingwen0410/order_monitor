訂單未交量報表產生器 — 使用說明
=====================================

【使用方式】
1. 雙擊外層的「訂單未交量報表產生器」捷徑
2. 在啟動視窗中確認或修改各項設定：
   - 公司別（ERP 資料庫）
   - 【訂單資訊管制表】路徑
   - 【生產記錄表】路徑
   - 報表儲存路徑
3. 點擊「執行」
4. 在「執行記錄」區域可即時查看處理進度
5. 完成後會自動開啟報表（可取消勾選）

【路徑記憶功能】
  程式會自動將您選擇的路徑記錄到 paths.json，
  下次開啟時會自動帶入上次的設定，無需重複選擇。

【輸出檔案】
  訂單狀態報表_YYYYMMDD.xlsx
  包含四個工作表：
    - 過期    ：客戶交期已到或超過今日的訂單（明細列出，依交期由舊到新排列）
    - 未過期  ：客戶交期尚未到期的訂單（明細列出，依交期由舊到新排列）
    - 客戶總表：各客戶的未交量總計、庫存差額一覽
    - 品號庫存：所有訂單分配完後的最終品號剩餘庫存總覽

【資料運算與扣庫存流程走向】
  1. 讀取 Excel：從 NAS 讀取訂單未交明細。
  2. 查核 ERP 庫存：至 ERP 資料庫即時抓取目前庫存量。
  3. 優先權排序：將所有訂單依照「品號 -> 狀態(過期優先) -> 交期(由舊至新)」進行排序。
  4. 庫存時序扣除：
     - 程式會嚴格依照上述的優先順序，一筆一筆地扣除庫存。
     - 若同個品號有多個客戶都在等待，最早交期的過期訂單會優先獲得庫存。
     - 在「過期 / 未過期」Sheet 中，您可以清楚看見庫存量「逐列遞減」的歷史軌跡。
     - 在「客戶總表」中，同客戶的過期與未過期狀態會分列顯示其對應的剩餘可用庫存。
  5. 產出報表：四個 Sheet 皆會自動加上篩選器，並調整欄寬。

【庫存差額欄說明】
  - 綠底 = 庫存量 > 未交量，庫存充足
  - 紅底 = 庫存量 <= 未交量，庫存不足
  - R&D 品號（如 TC-ROL, TC-PC）的庫存量顯示為「人員自行查找」，差額留空。

【設定檔說明】
  config.ini — 首次安裝時的預設路徑設定
  paths.json — 使用者上次選擇的路徑（自動產生，勿手動修改）

【問題排除】
  若執行出現錯誤，程式會跳出錯誤視窗，您也可至 %LOCALAPPDATA%\OrderMonitor 尋找 order_monitor.log，
  並將內容提供給 IT 人員協助排查。
